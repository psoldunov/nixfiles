# Rule File Templates

Phase 4 emits `.claude/rules/*.md` files in the project root. These mirror the user's global `~/.claude/rules/common/*.md` structure but are project-specific and derived from the findings.

## Principles

1. **Don't duplicate global rules.** If a project rule just restates something from the user's global rules (e.g., "write tests first"), omit it. Only emit project-specific guidance.
2. **Be concrete.** Every rule should reference a concrete file path or pattern the reader can verify.
3. **Explain the why.** Don't just say "use 2-space indent"; say "2-space indent (observed in 90% of TS files; matches `.prettierrc`)".
4. **Prefer observed over declared.** When config says X but code does Y, document Y.
5. **Skip empty categories.** If the findings for testing were "no tests present", don't generate `testing.md` — just mention it in the project summary.

## AGENTS.md pointer (with CLAUDE.md fallback)

After writing the rule files, ensure the project has an `AGENTS.md` at the root — this is the AI-agnostic convention shared across Cursor, Windsurf, Claude Code, and other tools. If `AGENTS.md` doesn't exist, create it with just the pointer. If it does exist, prepend:

```markdown
> Project rules live in `.claude/rules/` (generated by code-analyzer).
> Onboarding docs: [`docs/README.md`](./docs/README.md) — human-readable handbook.
> AI-agent rules: [`.claude/rules/`](./.claude/rules/) — terse, rule-shaped.
> Generated and kept in sync by the code-analyzer skill.
```

Do not duplicate the rule content in `AGENTS.md` — the files under `.claude/rules/` are loaded automatically by Claude Code and can be imported by other agents.

### CLAUDE.md backwards-compatibility shim

For tools that still look specifically for `CLAUDE.md`, create a minimal shim that redirects to `AGENTS.md`:

```markdown
> See [AGENTS.md](./AGENTS.md) for project instructions.
```

If an existing `CLAUDE.md` already contains real content (not already a pointer), **ask the user for confirmation** before migrating:
1. Move the existing content into `AGENTS.md` (prepend to any pointer already there).
2. Replace `CLAUDE.md` with the one-line pointer above.

Never silently overwrite a populated `CLAUDE.md` — its content may include hand-written instructions the user wants preserved in `AGENTS.md`.
